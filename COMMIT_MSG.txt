feat(wave-6 phases 6.4-6.6): passenger ETA, driver offline queue, real analytics + CSV

Phase 6.4 — Passenger ETA + favorites
  • New endpoint GET /api/routes/{code}/stops returns the route's stop
    sequence with per-stop eta_min (haversine from the closest in-service
    vehicle on the route, at the conservative 8-km/h floor). Uses the
    route_stops data that migration 015 seeded.
  • Passenger homepage popular-routes panel now shows real stops_count
    (no more "0 محطة"), surfaces favorited routes at the top, and has
    a star-toggle that persists to localStorage. No account required.

Phase 6.5 — Driver app polish
  • postPosition() now wraps fetch in a localStorage queue: failed or
    offline writes are buffered (cap 200), and a successful write OR
    the window.online event flushes them. The driver never loses a
    minute of GPS history because they ducked into a tunnel.

Phase 6.6 — Admin analytics + reporting
  • GET /api/admin/analytics/real?days=N — real Postgres-aggregated
    trips_by_day, completed total, passengers_total, alert types,
    occupancy buckets, open/critical alert counts, all operator-scoped.
  • GET /api/admin/analytics/drivers?days=N — per-driver scorecard:
    trips_completed, passengers_total, distance_km, incidents reported.
    Names are batched-hydrated in one users query.
  • GET /api/admin/export/{vehicles|users|trips|alerts|audit_log} —
    CSV download with operator scoping and the obvious role gates
    (users + audit_log require admin role, the rest are dispatcher+).
  • /dashboard/analytics.html now prefers the new real endpoint and
    falls back to the legacy hardcoded one for older builds.

57 endpoints total. AST parses clean.
